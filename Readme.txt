	Проект ESearch-cистема выдачи результатов поиска из поисковых систем Google,Yandex,Bing.

 Эти поисковые системы предоставляют программные интерфейсы(API) для поиска по интернету,сайту и т.д.. Другие разработчики могут использовать эти интерфейсы в своей системе,получить результаты и обрабатывать результаты как хочет. 
Но,в большинстве из них есть свои ограничения для использования.

	Коротко об использованных сервисах

 Google предоставляет разработчикам Custom Search JSON API.Можно использовать для поиска по опреденному сайту и по всему интернету.Для использования требуется API key и Custom search engine ID.Есть ограничение ,не более 100 запросов в день бесплатно. 

 У Yandex есть сервис Яндекс.XML.Возвращает результаты запроса в формате Xml.Для использования  нужно получить key и user.Ключ связывается с определенным ip-адресом.Без подтверждения номера телефона,предоставляется  бесплатно 10 запросов в сутки.


Microsoft предоставляет сервис Bing API.Есть пробная версия на 7дней бесплатно.Ограничение 3000 транзакций в месяц.Результат запроса получается в формате Json.


	Различные требования для работы программы

Чтобы все api поиска  работали,требуется ключи и т.д.

В проекте ESearch есть файл web.config. Внутри файла в секции appSettings есть исходные данные(ключи и т.д.) ,которые используются для идентификации пользователя api поиска.Значение YandexSearchApiKey связано с определенным ip-адресом.Если запрос будет отправляться с другого ip-адреса,то работать не будет.Поэтому нужно поставить свое значение для этого параметра.Соответсвенно значение для YandexSearchApiUser тоже свое будет.

Значение других параметров можно оставить,но лучше свое значение.
